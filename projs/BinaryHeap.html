Binary Min-Heap

Introduction

Imagine a network host receiving packets at hundreds of thousands per second. Due to the nature of network communication, some packets might be delivered out of order. We know that the out-of-order packets are only within a span of N consecutive sequence numbers. The host needs to maintain a buffer of the last N received packets and retrieve packets from the buffer based on the sequence number (smallest sequence number first to ensure ordered delivery). The buffer should provide the ability to find and remove the packet with the smallest sequence number efficiently, specifically, in around lgN operations. In Computer Science, lgN simply means log N base 2. This problem presents an ideal application for a data structure known as a binary Min-Heap.

Height of a complete binary tree

If \( N \) is the number of nodes in the heap and \( h \) is the height of the tree, then the total number of nodes in a complete binary tree can be expressed as a geometric series:

\[ N = 2^0 + 2^1 + 2^2 + \dots + 2^h \]

This series sums to:

\[ N = 2^{h+1} - 1 \]

\[ \Rightarrow N + 1 = 2^{h+1} \]

\[ \Rightarrow h + 1 = \log_2(N + 1) \]

\[ \Rightarrow h = \log_2(N + 1) - 1 \]

This means that any operation on the heap that requires traversing a path from the root to a leaf (or vice versa) will take a constant multiple of \( \log_2(N) \) basic operations to complete.
