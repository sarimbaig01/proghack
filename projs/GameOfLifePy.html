Python Game of Life

What is the game of life?

Conway's Game of Life is a cellular automaton created by mathematician John Conway, where cells evolve on a grid based on initial configurations. In this Python version, we use Pygame to visualize these cell states in real-time.

Basic terminology

Cell: The basic unit in the grid, either alive or dead.

Grid: A two-dimensional grid where cells evolve.

Neighbor: Any of the eight cells around a given cell.

Alive: A cell state set to `1`.

Dead: A cell state set to `0`.

Initial Configuration: The starting state of cells in the grid.

Generation: Each time step in the game.

Survival: Condition where a cell stays alive in the next generation.

The rules

Cells evolve based on the B3/S23 rule: a dead cell with three neighbors becomes alive, and a live cell with two or three neighbors survives. Implement this in Python by checking neighbor counts in each generation.

  1. A cell with fewer than two neighbors dies (underpopulation).
  2. A cell with two or three neighbors survives.
  3. A cell with more than three neighbors dies (overcrowding).
  4. A dead cell with exactly three neighbors becomes alive (reproduction).

Further terminology

Age: Generations a cell has been alive.

Pattern: Arrangement of live and dead cells.

Oscillator: Pattern that repeats over generations.

Still Life: Stable, unchanging pattern.

Equilibrium: When the grid no longer changes.

Population: Total live cells in the grid.
