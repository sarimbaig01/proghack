Rotated Maximum

We know from binary search that we can find any key from a sorted vector in logarithmic time. This is done by repeatedly halving the size of the vector that needs to be searched in the next step after only looking at the central element in each step. The problem of finding the maximum in a sorted vector would be trivial as simply the last element could be returned. However, consider a vector whose contents have been rotated right an unknown number of times. How may you now find its maximum element in logarithmic time?

Note: When we say logarithmic time, we mean that the algorithm takes a total of a log n + b basic operations to complete (where a and b are constants).

Example inputs:

The vector [4, 5, 6, 7, 0, 1, 2] has been rotated 3 places to the right.
The vector [6, 7, 0, 1, 2, 4, 5] has been rotated 2 places to the right.
The vector [1, 2, 3, 4, 5] has not been rotated.
The vector [2, 3, 4, 5, 1] has been rotated 1 place to the right.
The vector [5, 1, 2, 3, 4] has been rotated 4 places to the right.
